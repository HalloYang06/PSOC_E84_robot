安装提示

  1. 用安卓手机浏览器打开本页，点击“下载 APK”。
  2. 系统提示未知来源时，允许当前浏览器安装应用。
  3. 安装完成后，桌面应用名应显示为“灵动康复 ArmControl”。

1.0.8 更新

  1. 内置云端 API：http://106.55.62.122:8011。
  2. 启动后读取 public-config 和 catalog，不再静默只显示本地预览数据。
  3. 未登录时显示登录面板；登录后用 Bearer token 读取 /api/rehab-arm/app/v1/me。
  4. 首页覆盖静态成功文案，显示真实后端门禁、设备、计划和硬件协议阻塞状态。
  5. 登录后读取 /api/rehab-arm/app/v1/me/workflow，显示康复工作流、下一步、动作队列和禁止项。
  6. 新增 /api/rehab-arm/app/v1/me/workflow/actions：手机端可执行后端允许的安全流程动作，例如开始训练记录、生成报告、记录复盘；危险动作会被后端拒绝。
  7. 新增真实康复历史面板：从 /me.care_timeline 渲染训练记录、报告、AI 草稿和离线证据，不再依赖本地静态历史卡片。
  8. 新增 Android 原生 Bluetooth Classic SPP 桥：计划同步后可把后端生成的 legacy_transport_frame.wire_text 写入已配对 M33 设备。
  9. 蓝牙发送只接受后端标记 sendable=true 的旧协议帧；App 不伪造 M33 ACK，也不释放急停。
  10. 新增 M33 旧协议回包上传：Android 收到 memory_ack、execute_ack、sensor 或 error 后会提交到后端作为证据。
  11. SPP 回包只证明传输/设备回包，不会自动把计划改成 m33_accepted；正式执行仍等待 M33 决策链路。
  12. 新增 Stitch 设计的“蓝牙调试 / 实机验证”页面：从设备页进入，可读取 Android 已配对 SPP 设备、连接/断开、显示最新后端 legacy_transport_frame、发送 backend-approved frame，并展示 TX/RX/API/sensor 证据。
  13. 内部测试账号：3245056131@qq.com / 1234。

安全边界：手机 App 是训练、设备绑定、肌电和计划同步界面。它不能发送 CAN 帧、原始电机指令、力矩/电流命令、M33 覆盖指令，也不能释放急停。1.0.8 已加入 Stitch 蓝牙调试 / 实机验证页、已配对设备列表、SPP 连接/断开、后端批准帧发送和 ACK/sensor 证据显示；当前 M33 固件兼容性和真实硬件 ACK 仍需实测确认。
