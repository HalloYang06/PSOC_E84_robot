1.0.11 更新

  1. 新增社区内测页，底部“社区”不再是死链，用户可看到真实开放状态并跳转问康复师。
  2. 消息页接入后端康复师 Agent 接口 /api/rehab-arm/app/v1/agent/messages，服务端记录 client_type/purpose/scope，危险运动请求会安全拒绝。
  3. 设备页兼容 Android 原生 RehabArmSpp 桥，可读取已配对 Classic SPP 设备并继续账号绑定；浏览器环境会明确提示需要 Android App 蓝牙桥。
  4. 我的页设置按钮和设备页返回按钮补齐反馈/跳转，内测短信文案显示“内测验证码”。
  5. 我的页修正手机号绑定流程：未登录时手机号绑定区默认锁定，先登录云端账号后才可获取验证码和绑定。
  6. 云端 API 请求统一落到 8011 服务，避免登录和手机号绑定误打静态站。
  7. 设备绑定增加资源隔离：未登录时不会触发蓝牙权限、扫描或绑定接口，避免占用设备和账号资源。
  8. 内置云端 API：http://106.55.62.122:8011。
  9. 启动后读取 public-config 和 catalog，不再静默只显示本地预览数据。
  10. 未登录时显示登录面板；登录后用 Bearer token 读取 /api/rehab-arm/app/v1/me。
  11. 首页覆盖静态成功文案，显示真实后端门禁、设备、计划和硬件协议阻塞状态。
  12. 登录后读取 /api/rehab-arm/app/v1/me/workflow，显示康复工作流、下一步、动作队列和禁止项。
  13. 新增 /api/rehab-arm/app/v1/me/workflow/actions：手机端可执行后端允许的安全流程动作，例如开始训练记录、生成报告、记录复盘；危险动作会被后端拒绝。
  14. 新增真实康复历史面板：从 /me.care_timeline 渲染训练记录、报告、AI 草稿和离线证据，不再依赖本地静态历史卡片。
  15. 新增 Android 原生 Bluetooth Classic SPP 桥：计划同步后可把后端生成的 legacy_transport_frame.wire_text 写入已配对 M33 设备。
  16. 蓝牙发送只接受后端标记 sendable=true 的旧协议帧；App 不伪造 M33 ACK，也不释放急停。
  17. 新增 M33 旧协议回包上传：Android 收到 memory_ack、execute_ack、sensor 或 error 后会提交到后端作为证据。
  18. SPP 回包只证明传输/设备回包，不会自动把计划改成 m33_accepted；正式执行仍等待 M33 决策链路。
  19. 新增 Stitch 设计的“蓝牙调试 / 实机验证”页面：从设备页进入，可读取 Android 已配对 SPP 设备、连接/断开、显示最新后端 legacy_transport_frame、发送 backend-approved frame，并展示 TX/RX/API/sensor 证据。
  20. 新增已配对 SPP 设备一键绑定后端可信设备：实机验证页可以先把 Android 读到的 MAC/设备名写入 /devices/bind，再继续计划同步和 SPP 发送。
  21. AI 规划页已接入后端 /ai-training-drafts/generate 和 /accept：生成草稿、显示模型调用/规则回退状态，接受后落成训练计划；执行前仍需设备同步、M33 accepted 和 preflight。
  22. 训练库已绑定后端真实训练计划：AI 草稿接受后会在训练卡显示计划标题、目标角度、助力等级和 待同步 状态；点击未同步计划会进入设备同步页。
  23. 训练库显示后端康复历史，设备页和 AI 页保持聚焦，不再被全局历史面板挤占。

安全边界：手机 App 是训练、设备绑定、肌电、AI 草稿和计划同步界面。它不能发送 CAN 帧、原始电机指令、力矩/电流命令、M33 覆盖指令，也不能释放急停。1.0.11 已补齐社区入口、康复师 Agent 后端中转、设备页 Android SPP 桥兼容和内测验证码文案；当前 M33 固件兼容性和真实硬件 ACK 仍需实测确认。
