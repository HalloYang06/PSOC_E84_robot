## 人类视觉体验判断

桌面端整体可用，已经像一个真实平台，不是纯 demo。项目列表、项目主页面、NPC 工作台、观测台、AI 实验室都能承接真实数据，用户可以看到项目、成员、任务、审核、Runner、工位、回执、Boss Plan 和知识能力。

项目列表体验最好。它能清楚回答“我是谁、我有哪些项目、哪个项目需要处理、下一步是什么”。桌面和手机都没有横向溢出，手机首屏也能正常阅读。

项目主页面桌面端偏工程控制台。它的信息密度高，但核心状态能看懂：服务连接、当前任务、待人工审核、Runner 心跳、逻辑工位链路和右侧功能栏都在同一屏。适合熟悉平台的维护者，不太适合第一次进入的新用户。

NPC 工作台桌面端可用。左侧成员、顶部导航、Boss 区、资源索引、协作总览、派工状态都能理解。但它更像运维台，而不是轻量聊天工作台。

AI 实验室桌面端结构比较清晰。它把“实验运行、指标、训练数据、放行建议”拆成了工程师能理解的区域，也明确了 AI 不替人选择模型或发布。

观测台桌面端方向正确。它能把当前卡点、最近证据、派工验真和下一步动作整理出来，符合“负责人先看问题再决定”的产品目标。

手机端只有项目列表和部分项目主页面勉强可用。NPC 工作台和观测台在手机上不能作为正式体验交付。

## 问题清单

1. 移动端 NPC 工作台严重不可用。

   证据：`artifacts/visual-check/workbench-mobile-390x844.png`

   现象：标题和正文被压成竖排，右侧内容被裁切，布局没有折叠成单列。人类在手机上基本无法操作。

2. 移动端观测台存在横向溢出。

   证据：`artifacts/visual-check/observability-mobile-390x844.png` 和 `artifacts/visual-check/visual-results.json`

   自动检测结果：`overflowX=true`，多个元素宽度超过 390px，例如“当前目标 / 链路”、证据链卡片和部分主体卡片。

3. 桌面主页面信息密度过高，新用户会迷路。

   证据：`artifacts/visual-check/project-2d-1440x1000.png`

   现象：顶部导航、服务状态、资源中心、当前任务、人工审核、AI 线程、逻辑工位、右侧功能栏和流水线同时出现。熟悉系统的人能用，但第一次进入的人很难判断先点哪里。

4. “人工审核”下一步入口不够突出。

   证据：项目列表和项目主页面都显示有 1 条人工审核，但主操作按钮没有直接写成“处理这条审核”。

   影响：用户知道有风险门，却不一定知道从哪个模块进入处理。

5. 页面命名体系不统一。

   当前同时出现：`项目主页面`、`2D 开发者模式`、`驾驶舱`、`NPC 工作台`、`工作台`、`主页面`、`地图`。

   影响：新用户会怀疑这些是不是不同产品入口，尤其在顶部导航和返回按钮之间容易迷路。

6. 桌面端视觉风格过度偏“控制台”，缺少主次层级。

   现象：大量小字、边框、状态 chip、功能按钮同时出现。对工程师是高密度，对普通项目负责人是压迫感。

   建议：首屏只保留“当前目标、待处理、下一步、最近证据、主要入口”，其他诊断进入抽屉或二级页。

7. Runner 结果状态容易误用。

   现象：用常见词 `completed` 提交 Runner 结果返回 `INVALID_TASK_STATUS`，改用后端合法状态 `reviewing` 后通过。

   影响：接 Runner 的人很可能自然使用 completed/succeeded/done 等词。需要在 Runner 接入命令、接口文档或 UI 上明确合法状态。

8. 工位/线程/NPC 的概念对新用户仍偏抽象。

   现象：主页面说“逻辑工位、线程工位、NPC、Runner、电脑节点”，但缺少一个非常直接的解释或引导式创建顺序。

   建议：在新项目空状态中给出“一键推荐：创建 Boss NPC -> 创建电脑节点 -> 生成配对令牌 -> 创建线程工位 -> 派第一个任务”。

9. 本地工程验证没有完成。

   结果：`python -m pytest tests -q` 和 `npm run build:web` 在本轮执行时都超时，没有得到完整通过/失败结论。

   注意：这不影响线上黑盒验证结果，但影响代码层发布信心。后续需要延长超时时间或拆分测试集继续跑。

10. 截图暴露移动端部分页面不适合作为手机生产入口。

   结论：如果当前目标用户主要在桌面使用，可以先标注“移动端仅支持项目列表和简单查看”；如果要面向手机使用，必须先修复工作台和观测台的响应式布局。

## 建议修复顺序

1. 先修移动端 NPC 工作台，把两栏/三栏布局在 768px 以下改成单列，顶部项目名禁止竖排，导航允许横向滚动或折叠成菜单。
2. 修移动端观测台横向溢出，把固定宽卡片和宽 grid 改成 `minmax(0, 1fr)`，手机端所有卡片 `width: 100%`。
3. 在项目列表和主页面加一个强主按钮：`处理人工审核`，直接跳到对应审核区。
4. 统一入口命名，建议固定为：`项目列表`、`项目控制台`、`NPC 工作台`、`观测台`、`AI 实验室`。
5. 给新项目首屏增加引导式创建顺序，减少“工位/线程/电脑/NPC”概念同时砸给用户。
6. 更新 Runner 接入文档，明确任务结果合法状态，或者后端兼容 `completed -> reviewing/done`。
7. 拆分并补跑本地测试和前端构建。
