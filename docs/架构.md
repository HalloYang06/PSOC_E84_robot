  核心组件

  1. CAN Node (C++)

  功能：
  - 通过SocketCAN接口读写CAN消息
  - 发布CAN数据到ROS topic
  - 订阅ROS topic发送CAN命令

  关键技术：
  - ros2_socketcan 或直接使用 Linux SocketCAN API
  - 10ms定时器 (100Hz)
  - 异步I/O (Boost.Asio)

  2. WebSocket Bridge (C++)

  功能：
  - WebSocket服务器，监听客户端连接
  - 订阅CAN数据topic，通过WS发送给服务器
  - 接收WS消息，发布到ROS topic供CAN节点使用

  关键技术：
  - Boost.Beast WebSocket异步服务器
  - MultiThreadedExecutor
  - message_filters 同步多个topic

  3. Python辅助脚本

  功能：
  - 监控节点状态
  - 数据记录和可视化
  - 配置管理

  # ROS 2 依赖

    ros-jazzy-message-filters

  # CAN 工具
  sudo apt install can-utils

  sudo apt install libboost-all-dev

  pip3 install websockets rclpy

  CAN硬件配置

  # 加载CAN内核模块
  sudo modprobe can
  sudo modprobe can_raw
  sudo modprobe mcp251x  # 根据您的硬件芯片

  # 配置CAN接口 (500kbps波特率)
  sudo ip link set can0 type can bitrate 500000
  sudo ip link set can0 up

  # 验证
  ip link show can0
  candump can0  # 查看CAN消息

  关键代码框架

  CAN Node (伪代码)

  class CanNode : public rclcpp::Node {
    rclcpp::Publisher<can_msgs::msg::CanFrame>::SharedPtr pub_;
    rclcpp::Subscription<can_msgs::msg::CanFrame>::SharedPtr sub_;
    rclcpp::TimerBase::SharedPtr timer_;
    int socket_fd_;

    void timer_callback() {
      // 读取CAN消息
      struct can_frame frame;
      read(socket_fd_, &frame, sizeof(frame));

      // 发布到ROS
      auto msg = can_msgs::msg::CanFrame();
      msg.can_id = frame.can_id;
      // ... 填充数据
      pub_->publish(msg);
    }

    void can_cmd_callback(const can_msgs::msg::CanFrame::SharedPtr msg) {
      // 发送CAN消息
      struct can_frame frame;
      frame.can_id = msg->can_id;
      // ... 填充数据
      write(socket_fd_, &frame, sizeof(frame));
    }
  };

  WebSocket Bridge (伪代码)

  class WsBridge : public rclcpp::Node {
    boost::asio::io_context ioc_;
    websocket::stream<tcp::socket> ws_;
    rclcpp::Subscription<can_msgs::msg::CanFrame>::SharedPtr sub_;
    rclcpp::Publisher<can_msgs::msg::CanFrame>::SharedPtr pub_;

    void on_can_data(const can_msgs::msg::CanFrame::SharedPtr msg) {
      // 序列化为JSON
      json j = {{"id", msg->can_id}, {"data", msg->data}};

      // 异步发送WebSocket
      ws_.async_write(buffer(j.dump()),
        [](boost::system::error_code ec, std::size_t) {});
    }

    void on_ws_message(std::string msg) {
      // 解析JSON
      json j = json::parse(msg);

      // 发布到ROS
      auto can_msg = can_msgs::msg::CanFrame();
      can_msg.can_id = j["id"];
      // ... 填充数据
      pub_->publish(can_msg);
    }
  };

  性能优化要点

  1. QoS配置
    - CAN数据：RELIABLE + KEEP_LAST(10)
    - WebSocket：BEST_EFFORT + KEEP_LAST(1)
  2. 线程管理
    - CAN I/O：独立线程，SCHED_FIFO优先级
    - WebSocket I/O：Boost.Asio线程池
    - ROS Executor：MultiThreadedExecutor (2-4线程)
  3. 零拷贝通信
    - 启用intra_process_comms
    - 使用shared_ptr传递消息
  4. CPU亲和性
  cpu_set_t cpuset;
  CPU_ZERO(&cpuset);
  CPU_SET(2, &cpuset);  // 绑定到CPU2
  pthread_setaffinity_np(pthread_self(), sizeof(cpuset), &cpuset);

  启动流程

  # 1. 配置CAN接口
  sudo ip link set can0 up type can bitrate 500000

  # 2. 编译工作空间
  cd ~/ros2_ws
  colcon build --symlink-install

  # 3. Source环境
  source install/setup.bash

  # 4. 启动节点
  ros2 launch can_interface can.launch.py
  ros2 launch websocket_bridge bridge.launch.py

  # 5. 监控
  ros2 topic hz /can_rx  # 检查频率
  ros2 topic echo /can_rx  # 查看数据
