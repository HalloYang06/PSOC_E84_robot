# AI 线程汇报模板

用途：

给所有其他 AI 线程统一使用。每次完成一轮工作后，直接按这个模板写进自己的 `docs/ai-handoffs/<role>.md`，不要自由发挥。

---

## 标准模板

```md
## 本轮更新

- 当前阶段：
- 已完成：
- 修改文件：
- 当前验证：
- 截图路径：
- 当前分支：
- 下一步：
- 需要他人接力：
- 风险：
```

---

## 填写规则

### `当前阶段`

只能填以下 4 类之一：

- `进行中`
- `待验收`
- `待接手`
- `阻塞`

### `已完成`

要求：

1. 写真实完成的事，不要写打算。
2. 不能写空话，比如“优化了一些”“调整了若干”。
3. 必须对应到真实文件或真实结果。

### `修改文件`

要求：

1. 写相对明确的路径。
2. 只写本轮真正改过的文件。
3. 不要把没动过的文件写进去。

### `当前验证`

要求：

1. 说明自己如何验证。
2. 如果没法验证，要写清楚为什么。
3. 截图验证、编译验证、页面验证都可以，但必须诚实。

### `截图路径`

要求：

1. 有 UI 变化就必须写。
2. 没截图就明确写 `无`，并说明原因。
3. 截图最好放在 `artifacts/` 下。

### `当前分支`

要求：

1. 必须写当前工作分支。
2. 不允许省略。
3. 不允许在 `main` 上开发。

### `下一步`

要求：

1. 只写最自然的下一步。
2. 不要列一堆愿望清单。
3. 应该是自己范围内默认可以自动继续的动作。

### `需要他人接力`

要求：

1. 只有真需要别人接的时候才写。
2. 明确写“谁来接”“接什么”。
3. 不要写空泛的“后续继续完善”。

### `风险`

要求：

1. 风险必须具体。
2. 可以是代码风险、验证风险、方向风险、依赖风险。
3. 没风险时可以写 `暂无明确风险`，但不能空着。

---

## 不合格示例

```md
## 本轮更新

- 当前阶段：做完了
- 已完成：优化了很多
- 修改文件：若干文件
- 当前验证：看起来可以
- 截图路径：无
- 当前分支：不知道
- 下一步：继续优化
- 需要他人接力：大家一起看
- 风险：应该没有
```

这类汇报一律视为无效。

---

## 合格示例

```md
## 本轮更新

- 当前阶段：进行中
- 已完成：把任务田块从单任务推进改成并行产线；新增田块成长状态与资源消耗联动
- 修改文件：apps/web/public/harvest-moon-phaser3-game/index.html；docs/ai-handoffs/game-loop-core.md
- 当前验证：本地打开项目页后手动走完一轮“播种 -> 成长 -> 收获”；截图确认状态变化正常
- 截图路径：artifacts/game-loop-field-parallel.png
- 当前分支：ai/game-loop-core
- 下一步：继续补多田块并行时的资源稀缺冲突
- 需要他人接力：AI-3 需要根据当前并行田块逻辑补经济数值
- 风险：真实项目路由仍受 login 页面语法问题影响，当前验证主要依赖静态页面
```

---

## Boss 使用方式

我作为 boss 收到其他 AI 的结果时，优先看：

1. 当前阶段
2. 已完成
3. 当前验证
4. 截图路径
5. 风险

如果这 5 项不清楚，就视为还没有完成可交接状态。
