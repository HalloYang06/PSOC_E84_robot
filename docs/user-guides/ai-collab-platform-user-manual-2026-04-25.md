# AI 合作平台用户使用手册

更新时间：2026-04-25

适用对象：
- 第一次使用平台的项目负责人
- 需要邀请协作者一起接电脑、绑线程、创建 NPC、分发任务的用户
- 需要在同一张地图里进行多人、多电脑协作的团队

当前默认入口：
- 登录页：[http://127.0.0.1:3000/login](http://127.0.0.1:3000/login)
- 项目入口页：[http://127.0.0.1:3000/projects](http://127.0.0.1:3000/projects)

---

## 一、先理解 4 个核心对象

### 1. 账号
- 一个账号对应一个主角
- 登录进入项目后，地图里会出现你的主角
- 你的主角名下可以接入多台电脑

### 2. 电脑
- 一台电脑就是一个真实或虚拟工作节点
- 每台电脑下面可以挂多个线程
- 每台电脑都属于某个项目成员

### 3. 线程
- 线程是具体执行任务的 AI 会话
- 线程可以来自 `Codex`、`Claude`、`Qwen`
- 线程会回最小回执和最终回复

### 4. NPC
- NPC 是项目里的 AI 劳动力
- NPC 有职责、Skill、知识库、线程绑定
- 用户可以给 NPC 派任务，也可以让多个 NPC 顺序协作

## 二、首次进入平台

### 第一步：注册或登录

进入登录页后，输入邮箱、密码和名字即可。

![首次注册或登录](../../artifacts/dual-account-01-owner-signup-20260425-190419.png)

登录成功后，会进入项目入口页。

## 三、创建项目或接受邀请

### 方案 A：自己新建项目

如果你是项目发起人：
- 打开 `项目入口页`
- 进入 `新建项目`
- 填写项目名和简介
- 创建后会得到一个独立项目空间

![创建项目](../../artifacts/dual-account-02-owner-create-project-20260425-190419.png)

### 方案 B：接受别人邀请

如果你是协作者：
- 登录后打开 `接受邀请`
- 点击目标项目的接受按钮
- 接受成功后，项目会出现在你的项目列表里

![协作者接受邀请](../../artifacts/dual-account-06-member-invites-before-accept-20260425-190419.png)

---

## 四、邀请其他协作者加入项目

项目负责人可以在项目入口页邀请其他人：
- 打开 `邀请合作者`
- 填写对方邮箱
- 选择角色
- 提交邀请

![邀请协作者](../../artifacts/dual-account-04-owner-invite-third-20260425-190419.png)

建议做法：
- 一个真实玩家一个账号
- 一个账号一个主角
- 每个人用自己的账号加入同一项目

---

## 五、接入电脑

进入项目后，打开地图右侧的 `电脑接入管理`。

### 你可以做什么
- 添加一台电脑
- 给电脑生成配对令牌
- 扫描这台电脑上的线程
- 在项目里查看所有成员接入的电脑

### 添加电脑

点击 `添加电脑`，填写：
- 电脑 ID
- 电脑显示名
- 工作目录
- Git 目录

![Owner 创建电脑](../../artifacts/dual-account-07a-owner-create-computer-20260425-190419.png)

![第三位协作者创建电脑](../../artifacts/dual-account-07i-third-create-computer-20260425-190419.png)

### 查看电脑总览

电脑总览里会显示：
- 哪些玩家已接入
- 每个玩家有几台电脑
- 每台电脑下面有哪些线程

![电脑总览](../../artifacts/dual-account-07l-third-computers-overview-20260425-190419.png)

## 六、让线程进入平台

接完电脑后，需要给这台电脑配对并同步线程。

### 标准流程
1. 生成 `pairing token`
2. 用 token 注册 runner
3. 同步线程
4. 回到前端扫描线程

你在页面里会看到：
- 电脑已经绑定
- 线程已经出现在电脑下
- 其他项目成员也能看到这条线程

![生成配对令牌](../../artifacts/dual-account-07j-third-pairing-token-20260425-190419.png)

## 七、创建 NPC

进入 `NPC 管理` 后，点击 `添加 NPC`。

### 创建时建议填写
- 名字
- 职责
- 绑定哪台电脑
- 绑定哪条线程
- 选择合适的 Skill

这一步做完后，NPC 会成为可派工对象。

![创建 Writer NPC](../../artifacts/dual-account-08b-owner-create-writer-npc-20260425-190419.png)

## 八、给 NPC 派任务

进入 `协作消息池` 后，可以给 NPC 派任务。

当前推荐流程是：
1. 先预演
2. 再正式发送
3. 等最小回执
4. 等最终回复

### 示例：顺序协作
- 先给 `Research NPC` 派研究任务
- 它先回最小回执，再给最终回复
- 再把研究结果交给 `Writer NPC`
- `Writer NPC` 再完成收尾任务

![Research NPC 发单后回执](../../artifacts/dual-account-10b-owner-research-receipts-20260425-190419.png)

![Writer NPC 发单后回执](../../artifacts/dual-account-10e-owner-writer-receipts-20260425-190419.png)

## 十、协作现场怎么用

如果地图上看到其他玩家主角，可以点 `查看协作现场`。

进入后你可以看到：
- 当前共享动态
- 平台派工
- 最小回执
- 最终回复
- 线程焦点
- 高级证明

![第三位协作者进入协作现场](../../artifacts/dual-account-16e-third-owner-exchange-focus-20260425-190419.png)

### 二级区建议这样理解
- `成员动态`：谁最近在做什么
- `平台派工`：发给 NPC 或线程的任务
- `回执结果`：最小回执和最终回复
- `线程焦点`：当前该看哪条线程
- `高级证明`：更深的过程细节

### 三级抽屉

看到某条记录想看更多细节时，点 `查看详情`，会打开三级抽屉。

![第三位协作者打开详情抽屉](../../artifacts/dual-account-16g-third-exchange-detail-20260425-190419.png)

## 十二、推荐给新用户的标准使用顺序

如果你想最快上手，建议按下面顺序走：

1. 登录平台
2. 新建项目
3. 邀请协作者
4. 每个人接入自己的电脑
5. 每台电脑同步至少 1 条线程
6. 创建 1 到 2 个 NPC
7. 给 NPC 装 Skill
8. 在协作消息池里先预演，再正式派任务
9. 看最小回执和最终回复
10. 通过地图和协作现场继续跳转定位问题

## 十四、常见问题

### 1. 为什么我登录后看不到项目？
- 先确认你是否已经接受邀请
- 再确认当前登录的是正确账号
- 最后回到 `/projects` 刷新一次

### 2. 为什么我看不到别人的电脑？
- 先确认对方是不是在同一个项目
- 再确认对方有没有创建电脑并同步线程

### 3. 为什么 NPC 没回消息？
- 看一下协作消息池里是不是只预演了，还没正式发送
- 再看机房里的线程是不是在线
- 再看线程最近有没有最小回执或最终回复

### 4. 为什么地图里看不到别人的主角？
- 先确认对方是不是已经进入同一项目
- 再确认项目页是不是最新状态

## 十五、给项目负责人的建议

为了让多人协作不乱，建议你一开始就约定好：
- 一个账号对应一个真人玩家
- 一台电脑归一个玩家维护
- 一个 NPC 有清楚职责
- Skill 不要一开始装太多
- 任务尽量先从小闭环开始

推荐先跑一个最小链路：
- `Research NPC`
- `Writer NPC`
- 一条研究任务
- 一条收尾任务

这条链跑通后，再扩到更多 NPC、更多电脑、更多玩家。
