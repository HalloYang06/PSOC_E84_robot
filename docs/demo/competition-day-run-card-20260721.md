# 国赛现场一页操作卡

## 上场前只做这十项

1. 电脑连接设备专用路由器，确认本机出现 `192.168.3.x` 地址。
2. 确认云端 API 和控制台均返回 HTTP 200。
3. 确认 NanoPi 实际 IP，不依赖历史地址。
4. 确认 `rehab-rknpu-load` 与 `rehab-vla-vision` 为 active。
5. 确认左右摄像头是两个真实设备，水瓶连续可见 10 秒。
6. 确认 `192.168.3.34` 可达，MuJoCo 已打开并处于逻辑零点。
7. 确认 `can0` UP、ERROR-ACTIVE、非 LOOPBACK，并有新鲜 `0x322`。
8. 空载单独跑一次两点轨迹，确认 4/5/6 运动、停顿和回零。
9. 浏览器预加载总览、L、V、A、肌电、训练、日志，不再刷新页面。
10. 保底视频暂停在第一帧，安全观察员决定使用 L1/L2/L3/L4。

## 推荐档位

默认选择 **L2 混合闭环**：小智、平台、视觉、MuJoCo 实时，真机由操作员在 A 阶段触发预验证轨迹。

- 全部在线且完整预演通过：L1。
- 自动触发或坐标闭环不稳定：L2。
- CAN/M33/电机不满足：L3，只跑数字孪生。
- 网络或多设备不可用：L4，播放同步证据视频。

## 5 分钟提示

| 时间 | 动作 |
| --- | --- |
| 0:00-0:50 | 痛点、实物、双核与端云架构 |
| 0:50-2:20 | 说“我口渴了”，展示 L、V、MuJoCo和两点轨迹 |
| 2:20-2:50 | 说“帮我拿剪刀”，展示拒绝且机械臂不动 |
| 2:50-3:40 | 肌电波形、3D 肌肉模型、Int8 模型结果 |
| 3:40-4:20 | App 训练草案与审计，不把 App 说成遥控器 |
| 4:20-5:00 | 安全边界、创新点、下一步手眼标定与 6DOF |

## 两句固定语音

```text
小智，我口渴了。
小智，帮我拿剪刀。
```

第一句话后等待平台出现新 transcript 再切 V；第二句话后必须确认没有轨迹下发。

## 故障时只做一件事

| 故障 | 处理 |
| --- | --- |
| 页面打不开 | 5 秒内播放 VLA 保底视频 |
| 小智没同步 | 平台手工输入同一句话，切 L2 |
| 水瓶不稳定 | 固定瓶子，用最近一帧，不改阈值 |
| MuJoCo 不动 | 真机已单独验收则独立跑，否则切 L3 |
| CAN/M33 不满足 | 不发真机，切 L3 |
| 真机异常 | 立即停止，之后只演示数字孪生 |
| EMG 分类不稳 | 只讲实时波形和离线模型评估 |

## 必须说实话的边界

- 当前是可解释的 VLA-Lite，不是端到端大模型直接控制电机。
- 手眼标定尚未完成，相机 XYZ 不直接作为机械臂 base 坐标。
- 真机用预验证关节轨迹，视觉用于目标、距离和候选证据。
- M55、App、平台和 VLA 只给建议，M33 是最终安全裁决者。
