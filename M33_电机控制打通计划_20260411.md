## 0. 安全准备

第一次控制电机不要直接带机械负载。建议让电机空载或半悬空，确保人手、线缆、机械臂连杆都不在运动路径里。

上电前准备好这条急停命令，串口里随时可以发：

```text
cmd_motor_stop 1 1
```

其中 `1` 是关节号，不是 CAN 原始电机 ID。`clear_fault=1` 表示停机并尝试清故障。

建议先只接一个电机，也就是当前确认过的 `motor_id=0x01`。多电机不要一开始就一起测，避免 ID、方向、限位混在一起。

## 1. 初始化和身份确认

目标：确认烧录的是 direct PDL 版本，CAN 控制层可用，电机 ID 正确。

命令：

```text
cmd_control_init can0
cmd_motor_probe 0x01
cmd_motor_probe_last
```

预期输出：

```text
[control] init step6 direct pdl can init
[drv_can] direct ready pclk=20000000 nbtp=0x06000e03
control_init ret=0
[control] probe reply motor=0x01 uid=0x0137340c0230719f
MOTOR_PROBE: id=0x01 uid=0x0137340c0230719f
```

如果看到 `open can flags`，说明烧的是旧镜像，不要继续测控制。

如果 `cmd_motor_probe 0x01` 没回包，退回最小 PDL 链路确认：

```text
cmd_can_init_min
cmd_can_send_probe 0x01
cmd_can_poll_once
```

如果最小链路也收不到 `id=0x000001fe`，优先查接线、供电、收发器、波特率。

## 2. 只读参数验证

目标：在不让电机运动的情况下，确认“控制层发参数读帧、接参数回包、解析参数”的链路。

先读运行模式：

```text
cmd_motor_read 1 0x7005
cmd_motor_param
```

再读机械位置和机械速度：

```text
cmd_motor_read 1 0x7019
cmd_motor_param
cmd_motor_read 1 0x701B
cmd_motor_param
```

预期：`cmd_motor_read` 返回 `ret=0`，随后 `cmd_motor_param` 能打印对应 index 的回包。

如果 `ret=0` 但 `cmd_motor_param` 一直没有响应，说明发送成功但参数回包解析或电机响应还要继续查。此时不要进入运动控制。

## 3. 停止和使能验证

目标：确认最基础控制帧可发送，且电机没有报错。

先停机：

```text
cmd_motor_stop 1 1
```

预期：

```text
motor_stop ret=0
```

然后使能：

```text
cmd_motor_en 1
```

预期：

```text
motor_en ret=0
```

再读取反馈缓存：

```text
cmd_motor_fb 1
```

说明：`cmd_motor_fb` 显示的是最近一次反馈缓存。如果没有主动回报或还没有收到反馈，它可能显示旧值或 0 值。这个阶段主要看命令不报错，运动控制前仍然以“是否能停机”为第一优先级。

## 4. 速度模式小步测试

目标：用极小速度和电流限制验证速度模式，先正转，再停，再反转，再停。

推荐从非常小的值开始：

```text
cmd_motor_speed 1 0.2 0.5
```

含义：

```text
joint = 1
speed_rad_s = 0.2 rad/s
limit_cur = 0.5
```

运行 1 到 2 秒后立刻停：

```text
cmd_motor_stop 1 0
```

如果方向、声音、电流都正常，再测反向：

```text
cmd_motor_speed 1 -0.2 0.5
cmd_motor_stop 1 0
```

如果这一步不动，可以拆成低层参数写入来定位：

```text
cmd_motor_mode 1 2
cmd_motor_en 1
cmd_motor_write 1 0x7018 0.5
cmd_motor_write 1 0x700A 0.2
```

其中：

```text
0x7005 run_mode
0x7018 limit_cur
0x700A spd_ref
```

判断标准：

```text
ret=0 且电机低速平稳运动 -> 速度模式基本打通
ret=0 但完全不动 -> 查模式/使能/电机保护状态/参数单位
ret 非 0 -> 查发送链路或 direct PDL 返回值
运动方向相反 -> 先记录，后续再统一处理方向映射，不要马上乱改协议
```

## 5. 位置模式小步测试

目标：先用点到点位置模式 PP 做离散小角度移动，再考虑 CSP。

建议先使用 `csp=0`，也就是 PP 模式：

```text
cmd_motor_pos 1 0.05 0.2 0
```

含义：

```text
joint = 1
pos_rad = 0.05 rad
limit_spd = 0.2
csp = 0 -> PP 模式
```

观察到位后回到 0：

```text
cmd_motor_pos 1 0.00 0.2 0
```

再测负方向：

```text
cmd_motor_pos 1 -0.05 0.2 0
cmd_motor_pos 1 0.00 0.2 0
```

如果 PP 模式稳定，再试 CSP：

```text
cmd_motor_pos 1 0.03 0.2 1
cmd_motor_pos 1 0.00 0.2 1
```

注意：CSP 通常更适合上层周期性下发轨迹点。单条命令也许能动，但后面真正做轨迹时，应该用固定周期连续发送目标位置，而不是手敲单次命令。

## 6. MIT 私有控制帧测试

目标：验证 `cmd_motor_ctrl` 这条直接私有控制帧。这个阶段风险比速度/PP 更高，务必小增益、小目标、小力矩。

先进入 MIT 模式并使能：

```text
cmd_motor_mode 1 0
cmd_motor_en 1
```

先发保持命令：

```text
cmd_motor_ctrl 1 0.00 0.00 5.0 0.2 0.00
```

再小角度：

```text
cmd_motor_ctrl 1 0.03 0.00 5.0 0.2 0.00
cmd_motor_ctrl 1 0.00 0.00 5.0 0.2 0.00
```

参数含义：

```text
cmd_motor_ctrl <joint> <pos_rad> <vel_rad_s> <kp> <kd> <torque_nm>
```

当前配置范围：

```text
pos_rad: -12.57 ~ 12.57
vel_rad_s: -33.0 ~ 33.0
kp: 0.0 ~ 500.0
kd: 0.0 ~ 5.0
torque_nm: -14.0 ~ 14.0
```

建议开始值：

```text
kp = 5.0
kd = 0.2
torque_nm = 0.0
pos_rad 每次变化不超过 0.03 ~ 0.05
```

如果出现抖动、啸叫、冲击，立即：

```text
cmd_motor_stop 1 1
```

## 7. 多电机扩展计划

单电机 `0x01` 完成后，再逐个接入其他电机。不要一次接满。

每接一个电机，先确认 ID：

```text
cmd_motor_probe 0x02
cmd_motor_probe_last
```

如果真实 ID 和配置一致，再测：

```text
cmd_motor_stop 2 1
cmd_motor_en 2
cmd_motor_speed 2 0.2 0.5
cmd_motor_stop 2 0
```

当前默认 joint 到 motor_id 映射：

```text
joint 1 -> motor_id 0x01
joint 2 -> motor_id 0x02
joint 3 -> motor_id 0x03
joint 4 -> motor_id 0x04
joint 5 -> motor_id 0x05
```

如果实际电机 ID 不同，先改 `applications/control/control_layer_cfg.h` 里的 `CONTROL_MOTOR_JOINTx_ID`，不要在串口测试里强行混用。

## 8. 上层 ROS/CAN 指令接入

控制层已经有 ROS command CAN 入口，标准帧 ID：

```text
CONTROL_CAN_ID_ROS_COMMAND = 0x320
```

帧格式：

```text
Byte0: command
Byte1: joint_id
Byte2~7: payload
```

命令定义：

```text
0x01 enable
0x02 stop
0x03 set_target
0x04 set_mode
0x05 set_zero
0x06 active_report
```

`set_target` 解析方式：

```text
Byte2~3: target_pos_01deg, little-endian int16
Byte4~5: target_vel_rpm, little-endian int16
Byte6~7: target_torque_ma, little-endian int16
```

例子：使能 joint 1：

```text
STD ID 0x320
Data: 01 01 00 00 00 00 00 00
```

例子：停止 joint 1，不清故障：

```text
STD ID 0x320
Data: 02 01 00 00 00 00 00 00
```

例子：停止 joint 1，并清故障：

```text
STD ID 0x320
Data: 02 01 01 00 00 00 00 00
```

例子：设置 joint 1 目标，位置 1.0 度、速度 10 rpm、力矩 0 mA：

```text
target_pos_01deg = 10 -> 0x000A -> 0A 00
target_vel_rpm = 10 -> 0x000A -> 0A 00
target_torque_ma = 0 -> 0x0000 -> 00 00

STD ID 0x320
Data: 03 01 0A 00 0A 00 00 00
```

接 ROS 前，建议先用串口命令把速度/位置都跑通。否则上层一接入，问题会变成“到底是 CAN、电机、控制层、ROS 节点还是单位换算错”，排查难度会突然变大。

## 9. 建议后续补的工程命令

现在先不一定要马上改，但建议后续增加这些 MSH 命令：

```text
cmd_motor_active <joint> <0|1>
cmd_motor_zero <joint>
cmd_motor_stop_all [clear_fault]
cmd_motor_read_basic <joint>
```

目的：

```text
cmd_motor_active: 打开/关闭电机主动上报，方便 cmd_motor_fb 观察实时状态
cmd_motor_zero: 单独测试设零
cmd_motor_stop_all: 多电机阶段一键停所有电机
cmd_motor_read_basic: 一次读取 run_mode/mech_pos/mech_vel 等基础参数
```

优先级建议：

```text
第一优先级: cmd_motor_stop_all
第二优先级: cmd_motor_active
第三优先级: cmd_motor_read_basic
第四优先级: cmd_motor_zero
```

## 10. 推荐执行顺序清单

建议实际调试时按下面顺序一条条打钩：

```text
1. cmd_control_init can0
2. cmd_motor_probe 0x01
3. cmd_motor_probe_last
4. cmd_motor_read 1 0x7005
5. cmd_motor_param
6. cmd_motor_stop 1 1
7. cmd_motor_en 1
8. cmd_motor_speed 1 0.2 0.5
9. cmd_motor_stop 1 0
10. cmd_motor_speed 1 -0.2 0.5
11. cmd_motor_stop 1 0
12. cmd_motor_pos 1 0.05 0.2 0
13. cmd_motor_pos 1 0.00 0.2 0
14. cmd_motor_pos 1 -0.05 0.2 0
15. cmd_motor_pos 1 0.00 0.2 0
16. cmd_motor_mode 1 0
17. cmd_motor_en 1
18. cmd_motor_ctrl 1 0.00 0.00 5.0 0.2 0.00
19. cmd_motor_ctrl 1 0.03 0.00 5.0 0.2 0.00
20. cmd_motor_ctrl 1 0.00 0.00 5.0 0.2 0.00
21. cmd_motor_stop 1 1
```

每一步只要出现异常，就停在当前阶段，不要跳到下一阶段。尤其是速度和位置控制，宁可小步慢慢加，也不要一开始就给大速度、大位置、大增益。
