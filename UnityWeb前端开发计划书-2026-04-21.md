# Unity Web 前端开发计划书 - 2026-04-21

## 1. 项目目标
前端目标不是做一个普通管理后台，而是做一个可玩的 3D 生态科研基地。

玩家进入后应该能：
- 登录并进入项目
- 在 3D 基地里自由移动
- 看到 AI 对应的 NPC
- 与 NPC、建筑、设备交互
- 查看项目状态、电脑状态、任务状态
- 在世界里完成协作，而不是只点网页表单

## 2. 产品定位
前端定位为：
- Unity Web 3D 基地世界
- 轻量网页壳负责登录和项目切换
- 世界内交互优先
- 面板交互作为辅助

## 3. 页面结构
前端只保留 3 个页面层级：

### 3.1 登录注册页
职责：
- 登录
- 注册
- 找回当前项目入口

### 3.2 项目管理页
职责：
- 查看已有项目
- 创建新项目
- 接受邀请
- 选择进入某个项目

### 3.3 游戏页
职责：
- 加载 Unity WebGL
- 进入 3D 基地
- 打开团队面板
- 与 NPC 和建筑交互

## 4. 前端总体架构
建议采用双层结构：

### 4.1 网页外壳层
技术建议：
- Next.js 或现有 Web 外壳
- 负责登录态、项目选择、路由
- 负责和后端 API 通信
- 负责把数据传给 Unity

### 4.2 Unity 世界层
技术建议：
- Unity LTS
- WebGL 构建
- 负责 3D 世界、NPC、建筑、移动、交互、UI 提示

## 5. 世界设计方向
世界风格采用：
- 生态科研基地
- 接近自然
- 绿色养眼
- 未来感但不冰冷

空间建议：
- 中央生态中庭
- AI 协作温室
- 电脑机房
- 任务调度区
- 审批露台
- 机器人实验区
- 户外测试草坪

## 6. 核心玩法目标
前端第一阶段只做最小可玩闭环：

### 6.1 玩家能力
- 移动
- 转向
- 靠近交互
- 进入建筑
- 打开轻量面板

### 6.2 NPC 能力
- 待机
- 巡逻
- 说话
- 显示状态
- 接收任务
- 返回结果

### 6.3 建筑能力
- 开关门
- 显示状态
- 进入内部空间
- 提供功能入口

## 7. NPC 设计计划
每个 NPC 对应一个 AI 成员。

每个 NPC 前端需要支持：
- 唯一名字
- 唯一职位
- 外观模型
- 状态动画
- 头顶标识
- 交互入口
- 所属电脑/线程信息

建议首版 NPC 数量：
- 核心 NPC：3 到 5 个
- 背景 NPC：3 到 8 个

## 8. UI 计划
UI 原则：
- 世界优先
- 面板轻量
- 不遮挡视野
- 不做后台式大卡片

### 8.1 常驻 UI
- 右上：回项目管理页
- 左上：项目名 / 当前阶段
- 底部：交互提示

### 8.2 轻量面板
- 团队面板
- 任务面板
- 电脑面板
- 回执面板

### 8.3 世界内 UI
- NPC 头顶名字
- 建筑功能牌
- 状态气泡
- 任务标记

## 9. 门与建筑交互计划
所有关键建筑都要支持门状态。

门状态：
- 可进入
- 锁定
- 审批中
- 忙碌中

门交互：
- 接近显示提示
- 按键开门
- 满足条件才进入

## 10. 电脑与线程的前端表现
前端要把“电脑接入”和“线程扫描”做成世界内可理解的交互。

建议：
- 每台电脑对应一个设备位
- 每个设备位可显示在线状态
- 扫描后显示线程列表
- 可把线程绑定到 NPC 席位

## 11. 性能计划
目标设备：
- 普通办公电脑
- 中低端台式机
- 浏览器运行

性能策略：
- 低模或中低模资源
- 烘焙光照优先
- 植物做距离裁剪
- 限制活跃 NPC 数量
- 分区加载
- 轻后处理

## 12. 开发阶段

### 第一阶段：基础前端壳
目标：
- 3 个页面跑通
- Unity Web 接入网页
- 进入基础场景

交付：
- 登录页
- 项目页
- 空基地场景

### 第二阶段：基础基地可玩
目标：
- 玩家移动
- 建筑进入
- 3 个 NPC
- 基础对话

交付：
- 中庭
- 1 到 2 个建筑
- 轻量 UI

### 第三阶段：AI 协作可视化
目标：
- NPC 绑定 AI 席位
- 显示电脑状态
- 显示线程状态
- 可派发任务

交付：
- 团队面板
- 电脑面板
- 任务状态提示

### 第四阶段：交互深化
目标：
- 开关门
- 更完整动画
- 回执状态变化
- 世界反馈更自然

### 第五阶段：视觉打磨
目标：
- 环境统一
- 光照统一
- UI 精简
- 游戏感增强

## 13. 角色分工建议

### 前端开发
- 网页壳
- Unity 场景
- UI 桥接

### 技术美术
- 模型整理
- 材质灯光
- 动画接入

### 交互设计
- 建筑交互
- NPC 对话流
- 状态提示

## 14. 验收标准
前端第一版验收至少满足：
- 3 个页面结构清楚
- Unity 世界可进入
- 玩家能移动
- NPC 可见可交互
- 可进入建筑
- 可看到 AI 和电脑状态
- 不卡成不可用

## 15. 风险
- 模型风格不统一会导致整体难看
- 过度追求大世界会导致性能崩
- UI 太重会再次变成后台壳
- NPC 数量过多会拖慢浏览器体验

## 16. 一句话执行原则
前端先做“小而完整的生态科研基地”，不要一上来做大而乱的开放世界。
