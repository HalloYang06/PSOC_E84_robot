# AI 协作平台游戏化研发基地前端详细设计

版本：v0.1  
用途：把平台首页从普通 dashboard 设计成“研发基地”式工作台  
核心叙事：人类是总工程师，AI 是可管理的研发小队

---

## 1. 设计目标

传统后台首页通常是表格、图表和菜单，但本平台更适合做成一个“研发基地”：

```text
用户进入平台时，不是进入一个管理后台，
而是进入自己的 AI 研发公司/机器人实验室。
```

这个界面既要有趣，也要有真实工程价值。

设计目标：

1. 一眼看懂项目当前状态。
2. 一眼看懂哪些 AI 正在工作。
3. 一眼看懂哪里阻塞、哪里有风险。
4. 一眼看懂 token 和服务器成本。
5. 一眼看懂哪些任务需要人类确认。
6. 把复杂开发流程做成可视化研发基地。
7. 方便宣传截图和演示。

首页不叫 `dashboard`，建议叫：

```text

```

页面路由：

```text

```

也可以保留：

```text
/dashboard -> 自动跳转到 /base
```

桌面端布局建议：

```text
┌─────────────────────────────────────────────────────────────┐
│ 顶部状态栏：项目名 / 当前分支 / 在线 AI / 预算 / 风险 / 时间 │
```

## 3. 顶部状态栏

顶部状态栏用于展示全局状态。

字段：

```text
项目名称
当前环境
当前主分支
在线 AI 数
在线 Runner 数
今日 token 消耗
预算使用率
高风险告警数
待人类确认数
Git 同步状态
```

示例：

```text
康复机械臂研发基地 | develop | AI 在线 12/18 | Runner 5/6 | 今日 token 38.6 元 | 预算 42% | 高风险 2 | 待确认 3
```

状态颜色：

```text
绿色: 正常
黄色: 有提醒
橙色: 需要处理
红色: 高风险
灰色: 离线或未知
```

点击行为：

```text
点击预算 -> 财务室
点击风险 -> 急救站
点击待确认 -> 老板办公室确认列表
点击 Runner -> Runner 页面
点击 Git 状态 -> 代码车间
```

## 4. 老板办公室

### 4.1 定位

老板办公室是“人类总工程师”的工作区，不是 AI-Boss 的工作区。

主要处理：

```text
项目总览
预算
风险
重大决策
人类确认点
发布/回滚确认
```

### 4.2 卡片字段

```text
当前项目
项目阶段
本周目标
今日完成任务
进行中任务
阻塞任务
待确认事项
最高风险
下一步建议
```

示例：

```text
老板办公室

项目：康复机械臂
阶段：MVP 联调
本周目标：打通 NanoPi -> ROS -> PC1 数据链路
今日完成：6
进行中：9
阻塞：2
待确认：3
最高风险：真实机械臂运动测试需要人工确认

建议：
1. 优先处理 ROS 编译失败
2. 确认 NanoPi 传感器接口契约
3. 暂停低优先级文档任务，避免 token 超预算
```

### 4.3 操作按钮

```text
查看待确认
批准计划
暂停全部 AI
生成日报
进入发布检查
执行回滚
```

危险按钮规则：

1. `暂停全部 AI` 需要二次确认。
2. `执行回滚` 需要选择 commit/tag。
3. `发布` 必须通过发布检查清单。
4. 涉及硬件的确认必须显示 H 等级。

## 6. AI 工位区

### 6.1 定位

AI 工位区展示每个 AI 员工的实时工作状态。

每个 AI 是一个“工位卡”，不是单纯表格行。

### 6.2 AI 工位卡字段

```text
AI 名称
职位
所属电脑/Runner
模型
Agent 类型
当前任务
职责模块
上下文健康
今日 token
任务成功率
当前状态
是否可接手任务
最近一次输出
```

示例：

```text
AI11 - NanoPi ROS 主负责人
职位：ROS 工程师
电脑：PC1
模型：gpt-5.4
当前任务：TASK-023
职责：ROS 节点、topic、运行状态
上下文：黄色 63%
今日 token：18,230
成功率：82%
状态：等待 AI10 回复需求
可接手：否
```

### 6.3 状态条

每个 AI 卡片包含：

```text
上下文条
token 预算条
任务负载条
可靠性条
风险灯
```

颜色规则：

```text
绿色: 正常
黄色: 上下文偏多
橙色: 连续失败或接近预算
红色: 高风险或必须交接
灰色: 离线
蓝色: 等待其他 AI
紫色: 等待人类确认
```

### 6.4 AI 动作

```text
查看详情
暂停 AI
分配任务
生成摘要
生成交接包
指定接手
查看 token
查看最近提交
查看需求沟通
```

### 6.5 AI 情绪/状态表现

可以轻度游戏化，但必须对应真实数据：

```text
专注中 = 正在执行且上下文健康
卡住了 = 任务阻塞
需要休息 = 上下文超过 70%
超载 = 上下文超过 85% 或 token 超预算
等回复 = 等待需求单回复
等老板 = 等待人类确认
掉线 = Runner 离线
```
