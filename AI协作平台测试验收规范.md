# AI 协作平台测试验收规范

版本：v0.1  
用途：统一测试类型、验收标准、回归清单和发布前检查

---

## 1. 测试原则

1. 每个任务必须说明测试情况。
2. 无法测试必须说明原因。
3. 权限、安全、硬件相关任务必须有专项测试。
4. 修复 bug 必须补充回归用例。
5. 发布前必须跑完整检查清单。

---

## 2. 测试类型

```text
unit              单元测试
api               API 测试
integration       集成测试
frontend          前端组件/页面测试
runner            Runner 执行测试
security          安全测试
permission        权限测试
e2e               端到端测试
manual            人工验收
hardware-manual   硬件人工验收
```

---

## 3. 基础验收清单

每个任务完成必须回答：

```text
[ ] 是否完成任务目标
[ ] 是否只修改职责范围内文件
[ ] 是否有测试
[ ] 是否有错误处理
[ ] 是否有审计日志
[ ] 是否涉及 token 或密钥
[ ] 是否涉及权限
[ ] 是否涉及硬件
[ ] 是否需要人工确认
[ ] 是否说明回滚方式
```

---

## 4. 发布前检查

```text
[ ] 前端 build 通过
[ ] 后端测试通过
[ ] API 文档更新
[ ] 数据库 migration 可执行
[ ] Runner 基础任务通过
[ ] Git 分支和 diff 功能正常
[ ] token 不在日志和前端暴露
[ ] 人工确认点正常
[ ] 硬件操作默认阻止
[ ] 回滚流程验证
[ ] 关键文档更新
```

---

## 5. 第一版 E2E 场景

必须能跑通：

```text
创建项目
  -> 添加 AI
  -> 注册 Runner
  -> 创建任务
  -> 创建分支
  -> Runner 执行
  -> 生成 diff
  -> 记录 token
  -> 审查
  -> 人工确认
  -> 合并或回滚
```

---

## 6. 硬件人工验收

硬件相关任务不能只看 AI 输出，必须有人类填写：

```text
设备名称
接线状态
电源状态
测量结果
串口日志
照片或视频
风险确认
是否允许下一步
```
