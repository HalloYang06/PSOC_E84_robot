# 蓝牙迁移清单

## 目标路线
- 控制器：CYW55512 / CYW55513
- 板级链路：HCI UART
- 中间件目标：BTSTACK
- 协议目标：Bluetooth Classic / RFCOMM / SPP

## 已完成步骤
### 第 1 步
- 增加 HCI transport 骨架。
- 启动阶段不再伪装成蓝牙已连接。
- 当前缺口定位为蓝牙中间件未接入。

### 第 2 步
- 增加 stack adapter 探测层。
- 将缺口细化为 middleware / porting / profile 三类。
- 增加 `bt_stack_probe_cmd` 用于运行时探测。

### 第 3 步
- 增加 board bridge。
- 统一暴露板级蓝牙能力：芯片、HCI UART、低功耗、host wake、device wake。
- 增加 `bt_board_bridge_dump` 用于确认板级信息。

### 第 4 步
- 增加 middleware 骨架层。
- 将中间件目标固定为 BTSTACK + SPP。
- 当前探测结果推进为 `BTSTACK source integration`。

### 第 5 步
- 增加 RT-Thread porting 骨架层。
- 增加 `bt_porting_dump` 用于查看 RTOS 侧缺口。
- 仍然保持当前顶层缺口为 `BTSTACK source integration`，因为真实 BTSTACK 源尚未纳入工程。

### 第 6 步
- 增加 BTSTACK source manifest 层：
  - `bt_source_manifest.h`
  - `bt_source_manifest.c`
- 将真实源接入缺口拆成四个固定组件：
  - `btstack-core`
  - `btstack-port`
  - `btstack-chipset-cyw55`
  - `btstack-rfcomm-spp`
- 增加 `bt_source_manifest_dump` 用于输出当前源集成清单。
- middleware 探测开始依赖 source manifest，而不是只返回笼统的 `BTSTACK source integration`。

### 第 7 步
- 增加 BTSTACK source layout 层：
  - `bt_source_layout.h`
  - `bt_source_layout.c`
- 将后续真实接入分成五个目录组：
  - `core`
  - `port`
  - `chipset`
  - `profiles`
  - `config`
- 增加 `bt_source_layout_dump` 用于输出预期目录映射。
- `bt_source_manifest_dump` 现在会同时显示 source layout 完整度。

### 第 8 步
- 创建 BTSTACK 模板目录：
  - `middleware/btstack_template/core`
  - `middleware/btstack_template/chipset/cyw55`
  - `middleware/btstack_template/profiles/rfcomm_spp`
  - `middleware/btstack_template/config`
  - `applications/m33/btstack_port`
- 在各模板目录中加入 `README.txt` 占位说明。
- 增加：
  - `bt_source_template.h`
  - `bt_source_template.c`
- 增加 `bt_source_template_dump`，用于输出“真实官方源以后该放到哪里”。

### 第 9 步
- 增加：
  - `bt_upstream_manifest.h`
  - `bt_upstream_manifest.c`
- 将未来需要从官方来源引入的模块固定成 upstream manifest：
  - `btstack-core`
  - `btstack-chipset-cyw55`
  - `btstack-port-rtthread`
  - `btstack-rfcomm-spp`
  - `bt-configurator-output`
- 每个模块固定：
  - 上游来源提示
  - 本地目标目录
  - 需要的最小文件类别
- 增加 `bt_upstream_manifest_dump` 用于查看“下一步到底要从官方拿什么”。

### 第 10 步
- 增加：
  - `bt_source_entry.h`
  - `bt_source_entry.c`
- 增加 `bt_source_entry_dump`，用于输出最小入口源占位映射。
- 为以下目录补齐最小入口源文件占位：
  - `middleware/btstack_template/core/btstack_core_entry.c`
  - `applications/m33/btstack_port/btstack_port_entry.c`
  - `middleware/btstack_template/chipset/cyw55/btstack_chipset_cyw55_entry.c`
  - `middleware/btstack_template/profiles/rfcomm_spp/btstack_rfcomm_spp_entry.c`
- 当前这些入口源只是稳定文件名和接入位置，不含真实官方协议实现。

### 第 11 步
- 增加：
  - `bt_import_plan.h`
  - `bt_import_plan.c`
- 增加 `bt_import_plan_dump`，固定后续真实官方源导入顺序：
  - 先 core
  - 再 chipset
  - 再 port
  - 再 profiles
  - 最后 config
- 每一步都固定了：
  - 导入目标
  - 最小导入范围
  - 完成后的预期状态

### 第 12 步
- 增加：
  - `bt_core_import.h`
  - `bt_core_import.c`
- 增加 `bt_core_import_dump`，只聚焦第一批真实 BTSTACK core 导入文件。
- 将 `core` 第一批真实导入清单固定为：
  - `hci.c`
  - `l2cap.c`
  - `rfcomm.c`
  - `sdp_client.c`
  - `sdp_server.c`
- 更新 `middleware/btstack_template/core/README.txt`，把第一批导入文件写死。
