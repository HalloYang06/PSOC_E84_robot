# M33/M55 小智延迟无动作观测与验收

日期：2026-07-17

## 1. 目标和安全边界

本阶段只观测一轮小智交互的阶段延迟，不下发康复模式，不修改 CAN、电机、LVGL 或控制状态。

- M55 在首次非零 `rt_device_write()` 完成后生成一条 `MSG_TYPE_VOICE_LATENCY`。
- M55 使用零等待 IPC 发布；IPC 未就绪、互斥锁忙或队列忙时立即失败，不阻塞语音线程。
- M33 只校验并保存最新快照；接收回调不打印、不分配内存、不访问 CAN 和控制模块。
- M33 Shell 命令 `m55qa_xz_latency` 只读，不会发起语音或运动动作。

## 3. 指标定义

`NA` 表示该阶段没有被可靠观测，不等于 0 ms。

| 输出字段 | 定义 |
| --- | --- |
| `wake_listen_ms` | 本地唤醒/手动轮次开始到进入监听 |
| `voice_stop_ms` | 最近语音活动到 stop 请求交付前 |
| `stop_stt_ms` | stop 请求交付前到服务器 STT 事件 |
| `stt_llm_ms` | STT 事件到 LLM 事件 |
| `llm_tts_ms` | LLM 事件到 TTS start 事件 |
| `tts_packet_ms` | TTS start 到首个成功进入本地 TTS 队列的音频包 |
| `packet_write_ms` | 上述音频包到首次非零设备写完成 |
| `speech_audio_ms` | 最近语音活动到首次非零设备写完成 |
| `wake_audio_ms` | 轮次开始到首次非零设备写完成 |

注意：`first_write` 是设备写调用完成时间，不是扬声器真实出声时间；`first_packet` 当前是首个成功入队的包，不是被丢弃包的网络到达时间。

## 5. 首轮硬件 smoke test

1. 同时冷启动 M33 和 M55，先不要进入主动、助力或抗阻模式。
2. 等待 Wi-Fi、小智、IPC 和 Shell 均稳定；不要连续唤醒。
3. M55 执行 `m55qa_tts_diag`，记录初始 `latency turn/fail/drop`。
4. 完成一次语音问答，等待 TTS 播放结束后至少 3 秒。
5. M55 再执行 `m55qa_tts_diag`。预期 `turn` 增加，`fail` 和 `drop` 不增加。
6. M33 执行 `m55qa_xz_latency`。预期 `received_count=1`、`accepted_count=1`、`invalid_count=0`、`stale_count=0`。
7. 保存四组阶段和 total 输出；出现 `NA` 时先定位缺少的服务器事件，不把它记为 0 ms。
8. 重复前先确保上一轮播放完成。初期每轮间隔至少 5 秒。

这组 smoke test 不发送运动命令，测试时电机应保持未使能状态。

## 6. 失败判定

- `fail` 增加：M55 latency IPC 零等待发布失败。语音线程不应卡住，但该轮不会重试。
- `drop` 增加：上一轮未完成、新旧事件顺序不成立或首次写无法归属。
- `invalid_count` 增加：M33 收到非法 flags、零轮次或无 VALID 标记。
- `stale_count` 增加：重复/逆序轮次，或 M55 独立重启后序号回零。
- M33 没有数据且 M55 `fail` 不增加：检查双核 IPC 消费线程和消息分发。

## 7. 当前不能作为正式基准的场景

1. 多轮重叠、重试和网络抖动：TTS start/音频包尚未使用服务器 `session_id` 绑定，本轮开始后到达的上一轮数据可能被误归属。
2. 只复位 M55：`turn_seq` 从 1 重新开始，M33 可能持续判旧；当前 smoke test 要求双核同启。
3. TTS 队列满：当前 `first_packet` 表示首个成功入队包，不能代表真实网络首包。
4. IPC 最后一轮发布失败：M33 看不到这一轮，只能从 M55 `fail` 计数判断。

在修复 session/epoch 归属前，只允许把“静置后单轮、双核同启”的结果用于链路连通性验证，不用于比赛材料中的 P50/P95/P99 结论。

## 8. 正式基准的后续门槛

- 将服务器 `session_id` 或明确的 server turn token 绑定到 STT/LLM/TTS/音频事件。
- 增加 M55 boot epoch，或由 M33 可靠识别 IPC peer 重启并重置去重状态。
- 将 queue busy、mutex busy、runtime not ready 分开计数。
- 增加 turn 重叠、旧包、独立重启、序号回绕和 queue-full 的可执行测试。
- 完成至少 100 轮无重叠测试，再统计 P50/P95/P99/max 和有效样本率。
