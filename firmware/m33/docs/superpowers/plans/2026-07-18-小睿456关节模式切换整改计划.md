# 小睿 4/5/6 关节模式切换整改计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**目标：** 让小睿和 App 只能请求助力/抗阻模式，且每次对 4、5、6 号关节组（mask `0x38`）整体生效，任何实际运动仍必须通过 M33 安全检查。

**架构：** M55 只将受信任的结构化语音意图转换为 IPC 高层请求；M33 使用有界 MQ 和独立工作线程消费，先检查 4/5/6 的反馈时效、协议、ID、标定、故障和运行模式，再交给现有模式管理器。内部 PASSIVE/零电流仅作为安全回退，不作为语音或 App 可选训练模式。

**技术栈：** RT-Thread、C/C++、M33/M55 共享内存 IPC、Python/host C 回归测试、SCons、OpenOCD。
