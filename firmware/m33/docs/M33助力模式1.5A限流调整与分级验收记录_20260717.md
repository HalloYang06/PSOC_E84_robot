# M33 助力模式 1.5 A 限流调整与分级验收记录

日期：2026-07-17

## 1. 问题说明

首次 `0x320 + mask=0x10` 真机测试为了低能量验证 CAN 收包、MQ 消费、模式进入和 PASSIVE 回退，临时使用了 `0.2 A` 限流。它只能证明执行链路已打通，不能作为肌无力用户的正式助力工作电流。

进一步检查发现，原固件有两层 `1.0 A` 限制：

```c
#define CONTROL_MOTOR_CURRENT_CONTROL_MAX_A (1.0f)
#define CONTROL_REHAB_ASSIST_LIMIT_CUR_A     (1.0f)
```

即使 Shell 输入 `rehab cfg assist_max 1.5`，参数清洗也会把它夹到 `1.0 A`，底层 current-mode 接口同样会拒绝超过 `1.0 A` 的命令。

## 2. 本次最小修改

提交：

```text
7f2410b34 fix(rehab): raise assist current ceiling to 1.5A
```

只修改两个配置值：

```c
#define CONTROL_MOTOR_CURRENT_CONTROL_MAX_A (1.5f)
#define CONTROL_REHAB_ASSIST_LIMIT_CUR_A     (1.5f)
```

本次没有修改：

- 助力方向、增益、最小电流和策略算法。
- `0.03 A/周期` 的助力电流斜率限制。
- 电机故障位、反馈超时和反馈二次检查。
- CAN heartbeat lease、PASSIVE 紧急锁存和 STOP 路径。
- 助力模式状态机、ROS MQ 和 NanoPi 协议。

## 3. 电机规格边界

5 号关节在当前工程中配置为 RobStride RS00。官方 RS00 说明书给出额定负载相电流约 `4.7 Apk`、最大负载相电流约 `15.5 Apk`，因此 `1.5 A` 低于电机标称额定电流。

官方说明书：<https://robstride.com/assets/product_manual_robStride00-05044b8c.pdf>

注意：固件写入的是 `iq_ref(0x7006)`，必须继续确认电机当前固件对该参数的 A/Arms/Apk 口径。电机规格允许不等于佩戴系统已完成安全验收；机构力臂、减速比、电源、传动间隙和患者承受力仍然可能成为更低的系统上限。

## 4. 自动验证

- 新增的精确配置断言在原 `1.0 A` 配置上先失败，修改后通过。
- 助力实时故障阻断和 ROS MQ 消费路径共 `18 passed`。
- 完整 SCons 交叉编译通过，生成 `build/rtthread.hex`。
- 烧录、原始地址 verify 和 XIP 地址 verify 均通过。

旧 `tools/test_rehab_mode_static.py` 在完成本次 1.5 A 断言后，仍会被当前工作树中与本修复无关的 `M33_XIAOZHI_MINIMAL_FRAMEWORK` 断言阻断。本次没有为此修改或回滚小智代码。

## 5. 上电无动作检查

新固件上电后的实测状态：

```text
rehab mode=passive
assist_max_x1000=1500
MOTOR[5] mode=0 fault=0x00
VOICE_PRECHECK result=PASS
current_x1000=0
```

这证明新上限已进入运行固件，且上电不会自动进入助力。这不等于 1.5 A 人工施力验收已完成。

## 6. 分级人工施力验收

不应从 `0.2 A` 直接跳到 `1.5 A` 进行佩戴测试。建议在无人佩戴、机构可随时扶住、急停可达的条件下按以下顺序验收：

1. `0.5 A`：确认方向正确，施力时电流与助力力矩同向，PASSIVE 后立即归零。
2. `1.0 A`：确认无振荡、无突跳、无过温和无新故障位。
3. `1.5 A`：确认策略输出可达目标上限但不超限，heartbeat 中断或 PASSIVE 指令均能停止。

每一级都必须记录：

```text
ros_id / parsed / enq / applied
mode / fresh / current_x1000 / limit_x1000 / sat
MOTOR[5] mode / fault / torque / velocity / temperature
PASSIVE 后的 current_x1000 和 motor mode
```

任意一项出现方向错误、明显振荡、异常噪声、故障位、反馈过期、电流超限或 PASSIVE 不归零，都应立即停止，不进入下一级。

## 7. 当前结论

`1.5 A` 软件上限、编译、烧录和上电无动作检查已完成。真正的助力能力与佩戴安全仍需要操作者在现场完成 `0.5 -> 1.0 -> 1.5 A` 人工施力验收，当前不应将其记录为已通过。
