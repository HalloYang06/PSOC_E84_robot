## 当前默认探测帧

`cmd_can_send_probe` 当前按 RobStride 私有 `Get_ID` 发扩展帧，格式为：

- `comm_type = 0x00`
- `master = 0xFD`
- `motor_id = 参数`

例如：

```text
cmd_can_send_probe 0x7F
```

生成扩展 ID：

```text
0x0000FD7F
```

## 下一步建议

不要再先碰 `cmd_control_init`。优先继续走最小链：

1. 反复测试最小初始化

```text
cmd_can_init_min
```

2. 逐个探测常见电机 ID

```text
cmd_can_send_probe
cmd_can_poll_once
cmd_can_send_probe 0x01
cmd_can_poll_once
cmd_can_send_probe 0x02
cmd_can_poll_once
cmd_can_send_probe 0x03
cmd_can_poll_once
cmd_can_send_probe 0x07
cmd_can_poll_once
cmd_can_send_probe 0x7F
cmd_can_poll_once
```

3. 如果仍无回包，优先检查硬件
- CANH/CANL 是否接对
- 终端电阻是否正确
- 电机是否上电
- 电机是否真的使用这套 RobStride 私有协议
- 波特率是否真是 `1Mbps`

4. 如果最小链收发通了，再回头重构 `control_layer`
- 不要直接复用当前 `cmd_control_init` 全链
- 先做轮询版
- 再做中断版
