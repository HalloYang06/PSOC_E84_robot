# 阶段4开发日志：联调与稳定性收敛

## 日期
- 2026-03-29

## 阶段目标
- 形成可执行的联调入口
- 固化现场验收检查点
- 让团队成员按同一流程复现结果

## 本阶段产出
- 脚本：`tools/can/phase4_joint_validation.sh`
  - 一键配置经典 CAN 1Mbps
  - 发送 F103 状态查询命令
  - 自动抓包并做基础规则检查（FD帧、7C1/7C2/7C3）
- 文档：`phase4_joint_test_checklist.md`
  - 覆盖总线、电机、F103、ROS、稳定性、性能六类检查项

## 收敛思路
- 先自动化检查“硬性约束”（经典 CAN / 关键ID帧可见）
- 再人工核对“系统级指标”（延迟、抖动、恢复能力）
- 把问题定位分层：总线层 -> 协议层 -> 应用层
