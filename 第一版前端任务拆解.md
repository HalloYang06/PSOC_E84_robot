## 1) 第一版前端页面清单

必做页面：

1. `/base`（或 `/dashboard` 跳转到 `/base`）：研发基地首页
2. `/projects`：项目列表
3. `/projects/:id`：项目详情
4. `/agents`：AI 成员列表
5. `/agents/:id`：AI 成员详情
6. `/runners`：Runner 状态
7. `/tasks`：任务列表
8. `/tasks/:id`：任务详情
9. `/tasks/:id/diff`：diff 审查
10. `/tasks/:id/logs`：日志查看
11. `/tasks/:id/context`：上下文健康
12. `/usage`：token 与成本简表
13. `/lab`：硬件实验室（人工确认与上传入口）

第一版可暂缓（不阻塞闭环）：

1. `/requirements`：需求中心（可先内嵌在任务详情里用简版）
2. `/git`：Git 分支/commit 独立页（可先在“代码车间”卡片摘要展示）
3. `/settings`：设置（第一版可先不开放或只做占位）

## 3) 组件拆分建议（按层次与复用）

建议按以下层次组织组件：

1. 页面级（路由组件）：只负责取数、组合面板
2. 面板级（业务组件）：BossOfficePanel / TaskHallPanel 等
3. 通用 UI（可复用）：Button / Dialog / Tabs / Badge / Table / Empty / Loading / Error
4. 指标组件（可复用）：ContextMeter / TokenBudgetMeter / StatusBadge / RiskBadge
5. 查看器组件（专业）：DiffViewer / LogsViewer

建议第一版就统一的通用能力：

1. `StatusBadge`：任务状态、AI 状态、Runner 状态
2. `RiskBadge`：H0-H4 风险等级 + 文本说明
3. `HumanApprovalDialog`：所有人工确认使用同一弹窗
4. `EmptyState`：空列表/未配置提示
5. `ErrorState`：统一错误展示
6. `LoadingState`：统一加载骨架

Mock/真实 API 切换建议：

1. `lib/api/*`：统一 API client
2. `lib/mock/*`：统一 mock 数据
3. 页面取数只调用 `services/*`，由 services 内部决定 mock 还是 API

## 4) 前端第一周执行顺序（建议）

Day 1：工程骨架

1. Next.js + TS + Tailwind 初始化
2. App Router 路由骨架（必做页面都建空壳）
3. Layout / 导航 / 主题基线
4. 通用组件基线：Button、Dialog、Badge、Empty、Loading、Error
5. 状态枚举与颜色映射（任务/AI/Runner/风险）

Day 2：研发基地首页骨架（先 mock）

1. `/base` 布局网格（桌面/移动）
2. `BaseTopBar`、`BossOfficePanel`、`TaskHallPanel`、`AgentWorkstationGrid` 基本可用
3. `HumanApprovalDialog` 统一弹窗完成

Day 3：管理页第一批（先 mock）

1. `/projects`、`/agents`、`/runners` 列表 + 详情基本展示
2. 创建/编辑表单的最小版本（必填 + 基础校验）

Day 4：任务链路与审查页（先 mock）

1. `/tasks`、`/tasks/:id` 任务详情
2. `/tasks/:id/diff` DiffViewer 简版
3. `/tasks/:id/logs` LogsViewer 简版（摘要/完整）

Day 5：上下文/成本/硬件安全（先 mock + 预留联调口）

1. `/tasks/:id/context` 上下文健康 + 交接包入口
2. `/usage` token 成本简表
3. `/lab` 硬件确认卡 + 上传入口 + H0-H4 展示
4. 与后端对齐 DTO 字段（若后端可用则开始替换 mock）

## 6) 移动端必须检查的点（第一版强制项）

通用检查：

1. 360px 宽度下不溢出、不横向滚动（除非日志/diff 查看器本身）
2. 关键按钮可点（尺寸足够，间距足够）
3. 状态不能只靠颜色表达，必须带文字（例如“红色=高风险”也要写出来）
4. 页面要有 loading/empty/error 三态
5. 长文本自动换行或折叠（任务标题、AI 名称、分支名）
6. 表单输入体验正常（键盘遮挡、滚动到可见区域）

研发基地 `/base`：

1. 顶部状态栏在手机端要简化展示，不压缩到看不清
2. “待人类确认”必须在首屏或首屏可见区域
3. AI 工位卡列表要可滚动，卡片高度稳定
4. 高风险事件（红色）必须强提示，不被卡片淹没

任务相关：

1. `/tasks/:id/diff`：diff 允许横向滚动，但必须可控；文件列表可折叠
2. `/tasks/:id/logs`：日志等宽字体，支持复制；错误高亮不影响阅读
3. `/tasks/:id/context`：上下文健康条在手机端必须可读

硬件实验室 `/lab`：

1. H3/H4 风险提示必须明显
2. 上传入口（日志/图片）可用
3. 明确禁止自动硬件操作的提示不可省略
