# AI 协作平台安全权限规范

版本：v0.1  
用途：统一用户权限、AI 权限、人工确认、密钥管理和硬件安全规则

---

## 1. 安全原则

1. 人类主导。
2. 最小权限。
3. 默认拒绝。
4. 高风险操作必须人工确认。
5. 明文密钥不得进入前端、日志和仓库。
6. 所有关键操作必须审计。
7. 真实硬件永远不能被 AI 自动控制。

---

## 2. AI 权限等级

```text
L0 只读:
  只能读取代码、文档、日志。

L1 建议:
  可以生成 patch 或建议，但不能写仓库。

L2 开发:
  可以创建任务分支、提交代码、运行测试。

L3 维护:
  可以创建 PR/MR、触发 CI、修改 issue。

L4 管理:
  可以回滚、修改任务分配、修改部分配置。

L5 硬件控制:
  理论权限，默认禁用，必须人类临时授权。
```

---

## 3. 人工确认等级

```text
H0 无需确认:
  文档摘要、代码解释、日志分类。

H1 轻确认:
  普通代码修改、测试脚本、README 更新。

H2 必须确认:
  合并 PR、修改接口协议、修改构建配置。

H3 双重确认:
  烧录固件、修改控制参数、访问真实设备。

H4 禁止自动执行:
  机械臂运动、电源操作、删除关键数据、绕过安全保护。
```

---

## 4. 密钥管理

规则：

1. 前端不显示明文 token。
2. 后端加密保存 token。
3. Runner 只拿临时凭证。
4. 任务结束清理凭证。
5. token 使用必须记录日志。
6. 支持 token 轮换。
7. 不允许 AI 在输出中打印密钥。

密钥类型：

```text
GitHub token
Git SSH key
模型 API key
Runner token
服务器 SSH key
数据库密码
对象存储密钥
设备调试凭证
```

---

## 5. 硬件安全

AI 可以：

```text
分析日志
查手册
生成检查清单
建议测量点
分析人类上传的测量结果
生成代码建议
```

AI 不可以自动：

```text
上电
烧录
控制电机
控制机械臂
修改急停逻辑
修改限位保护
绕过安全检查
```

---

## 6. 审计日志

必须记录：

```text
谁执行
执行时间
操作类型
关联项目
关联任务
影响对象
操作前状态
操作后状态
是否成功
失败原因
是否涉及权限/密钥/硬件
```
